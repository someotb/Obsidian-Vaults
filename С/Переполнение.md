Представим вымышленную переменную состоящую из 3-х бит:
Мы можем записать в нее 8 значений, согласно 2-ой системе:
- 000  - 0
- 001  - 1
- 010  - 2
- 011  - 3
- 100  - 4
- 101  - 5
- 110  - 6
- 111  - 7

Вымышленный тип данных "uint3_t", запертое пронстранство от 0 до 7. В математике такая алгебра называется "Кольцо вычетов по модулю 8"

Добавление 1 к числу 111 - 7, приведет к переполнению разряда и приведет нас к 0, так как прибавлять уже некуда, это называется:
- overflow - переполнение вверх (7 + 1 = 0)
- underflow - переполнение вниз (0 - 1 = 7)

Для кодирования отрицательных чисел, придумаем тип "int3_t" это будет так:
- 000  - 0
- 001  - 1
- 010  - 2
- 011  - 3
- 100  - (-4)
- 101  - (-3)
- 110  - (-2)
- 111  - (-1)
Не симметрия, половина "сдвинута" к отрицательным числам

Пример "overflow" на C:
```c
#include<stdio.h>
#include<inttypes.h>

int main()
{
    int8_t x = 127;
    printf("x = %d\n", x);
    x++;
    printf("x = %d\n", x);

    return 0;
}

/*
Вывод будет таким:
x = 127
x = -128
*/
```

Пример "underflow" на C:
```c
#include<stdio.h>
#include<inttypes.h>

int main()
{
    int8_t x = 127;
    printf("x = %d\n", x);
    x++;
    printf("x = %d\n", x);
    x--;
    printf("x = %d\n", x);
    return 0;
}

/*
Вывод будет таким:
x = 127
x = -128
x = 127
*/
```

Пример с умнажением:
```c
#include<stdio.h>
#include<inttypes.h>

int main()
{
    int8_t x = 127;
    printf("x = %d\n", x);
    x++;
    printf("x = %d\n", x);
    x *= 2;
    printf("x = %d\n", x);
    return 0;
}

/*
Вывод будет таким:
x = 127
x = -128
x = 0
*/
```
