Все типы данных в языке С++ подразделяются на три категории:
- ==Базовые типы== - это стандартные типы данных, предопределенные в самом языке. Их имена зафиксированы в самом стандарте и обозначаются ключевыми словами, которые никак кроме как для обозначения типа использовать нельзя. Базовые типы являются минимальными кирпичиками, на основе которых можно строить более сложные, пользовательские типы данных. В то же время базовые типы нельзя разложить на более простые.
- ==Произвольные типы== - это типы данных, создаваемые разработчиком при написании программы.
- ==Типы класса== - в рамках ООП каждый класс является типизирующим элементом. То есть каждый экземпляр класса(объект), используемый в программе, имеет тип по имени класса.

**Общее описание**
В языке С++ определено 7 базовых типов данных
- Символьный - char;
- Символьный двухбайтовый - wchar_t;
- Логический - bool;
- Вещественный числа двойной точности - double;
- Вещественные числа - float;
- Целые числа - int;
- Значение не возвращается(пустое значение) - void;

Совместно с именами типов могут использоваться модификаторы типа. ==*Модификаторы типa== - это ключевые слова, которые могут указываться перед именем базового типа и модифицировать его*. В языке С++ предусмотрено четыре модификатора типа:
- signed -значение со знаком;
- unsigned - значение без знака;
- short - укороченный тип;
- long - расширенный тип;
- long long - целевой тип будет иметь размер не меньше 64 бит(нововведение стандарта С++11)

**При использовании символьного типа, нужно обратить внимание на то, что символы должны быть заключены в одинарные кавычки**.

==Целочисленное значение== - это значит целое число, без десятичного разделителя. Например: 34 - это целое число, а 36.6 - это не целое(вещественное) число.

**В качестве десятичного разделителя в С++ используется символ "." - точка.****

**Тип void обычно используется для указания типа значения, возвращаемого функцией, когда эта функция ничего не возвращает. Согласно стандарту С++ для каждой функции мы должны указывать тип возвращаемого значения. Но далеко не все функции в результате своей работы должны что-то возвращать.**
